# .kiro/_archive

Documentos do Kiro a nível de workspace preservados como referência histórica.

## Conteúdo

- `AUTO_APPROVE_SETUP.md` — instruções de setup de auto-approve para sessões autônomas do Kiro. Não é mais necessário com a configuração atual; mantido como referência caso uma sessão futura precise reativar o modo.

## Diferença para `.kiro/specs/_archive/`

- `.kiro/_archive/` (este diretório) — documentos do **workspace** Kiro (configuração, setup, instruções operacionais).
- `.kiro/specs/_archive/` — **specs** arquivados (5 pastas pré-auditoria + 1 bugfix entregue), referenciadas como `historical_refs` pelas fases 1-7 da auditoria geral.

Os specs ativos pós-auditoria continuam em `.kiro/specs/fase-*/` e `.kiro/specs/auditoria-geral/` como referência viva.
